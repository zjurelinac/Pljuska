Language

Overview and documentation of a HASH scripting language

Introduction

Hash is a simple and easy-to-use scripting language. It's syntax and commands are quite similar to other UNIX scripting languages, so if you've ever used any of them, say Bash or Zsh, you'll find Hash familiar and understandable.

The documentation is divided in two parts. The first deals with basic language syntax and features, while the second contains a short description and usage examples of every command supported by Hash.

Language overview

Here is a short overview of Hash language characteristics and features. It's separated into smaller paragraphs, each describing one of the core elements of the language.

Syntax

Hash language script consists of a series of statements, which can be either commands, assignments or control statements. Statements are most often separated by newlines, though semicolons ; can serve the same purpose.

Here is an example of a short script:

# For each month, create a directory for notes
cd Documents/Bussiness
$i = 1
while [ $i <= 12 ]{
    mkdir "month_$i_notes"
    $i = $i + 1
}

Data types

Hash supports two separate data types: integers and strings, whereby string values must be enclosed into double quotes, "like this". Integer values support all the basic arithmetic operations: addition, subtraction, multiplication, division and modulo, while strings can contain variables whose values are inserted into it when the string is first used. Integer values are limited to the interval of <-231, 231—1>

Variables

Variables in Hash can be of either string or integer type, but it's type is changeable during program execution. Variables are always identified by preceding a $ symbol in front of their name. Variable names can contain both uppercase and lowercase letters, digits and an underscore symbol _.

Beware though, some variable names are reserved for special purposes, like $0 which contains the script name, and $1, $2..., where script arguments are automatically stored. Accidentally overriding their values could influence the behaviour of other functions.

Arithmetic

Integer data type in Hash supports all basic arithmetic operations: addition +, subtraction —, multiplication*, division / and modulo %. Their behaviour is the same as almost everywhere else: they're all associative, and multiplication, division and modulo have higher priority than addition and subtraction. There is also an unary - operator, which behaves as expected.

Unary minus should be enclosed in parentheses together with the term it applies on, because otherwise there is a possibility of parse errors.

You can also use parentheses to group certain terms and override operator precedence, should you require to do so.

A few examples of arithmetic expressions:

$root = ( $a + $n / $a ) / 2
$year = ( $days - $days / 1460 + $days / 36524 - $days / 146097 ) / 365

Conditions

Conditions in HASH are created from simple comparisons between values and variables, such as $a >= 7, which are combined using operators and &&, or || and not !, together with parentheses. They're allowed only inside control statements, where their evaluation to either True or False determines how will the script execution proceed.

An example of a condition inside an if:

if [ ( $len > 0 ) && ( $shouldOutput != "no" ) ] {
    # Do something accordingly
}

Control statements

Hash supports two types of control statements: if–else statements and while loops. Both test a given condition, and based upon it determine what to do next.
